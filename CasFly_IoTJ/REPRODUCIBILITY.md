# CasFly Reproducibility Notes

## Dataset used in the paper

The CasFly paper's **published experimental evaluation uses Synthea synthetic EHR data**.
It is not MIMIC-only.

Evidence from `2Casfly.pdf`:
- Section IV-A states they "adopt the Synthea synthetic EHR generator" for evaluation.
- Section IV-B states each device maintains local TPHG from "the Synthea dataset".
- The paper also notes Synthea was used for reproducibility and PHI safety.

In this repo:
- `main/csv/*` and `main/csv/device_partitions_patientwise1_rasp/*` are the Synthea-path artifacts used by the paper-style evaluation scripts.
- `main/mimic-iv-clinical-database-demo-2.2/*` and `main/code_mimic_all/*` are useful extensions for MIMIC-backed experiments, but that is separate from the Synthea-based paper section.

## One-command reproduction

From repo root (`/Users/anshita/PhD/sigmod`):

```bash
python3 -m pip install -r main/csv/requirements_repro.txt
bash main/reproduce_casfly_results.sh
```

Optional output directory:

```bash
bash main/reproduce_casfly_results.sh /absolute/path/to/output_dir
```

## What gets regenerated

Primary outputs are written under `main/csv/artifacts/repro_paper` (or your custom output path):

- `plots/`:
  - macro metrics vs tau
  - overlap bars
  - hits@K
  - complexity scatter
  - other paper-style plots generated by `pipeline_pcmci_vs_lave.py`
- `tables/`:
  - `macro_by_tau.csv`
  - `per_device_metrics_by_tau.csv`
  - `overlap_by_tau.csv`
  - `hits_at_k_by_tau.csv`
- `tau_*/`:
  - `edge_overlap_counts.csv`
  - `computed_metrics_from_overlap.csv`
  - `hits_at_k.csv`
- `addons/`:
  - predictive utility and paired tests (`summary_by_tau.csv`, `paired_tests.csv`, etc.)
  - reverse hits (`reverse_hits_by_tau.csv`)

## Determinism and reruns

- The script runs the tau sweep in `--resume` mode to reuse existing PCMCI artifacts in the output directory.
- To force clean regeneration, delete the chosen output directory and rerun.

## Notes

- `tigramite` is required for PCMCI+ stages.
- Runtime depends on machine and number of devices/taus.
